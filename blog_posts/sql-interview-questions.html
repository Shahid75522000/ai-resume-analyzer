Most Asked SQL Interview Questions (With Answers)

Prepare for Data Analyst, MIS, BI and Reporting interviews with these essential SQL questions and clear explanations.

SQL is one of the most important skills for Data Analysts, MIS Executives and BI Developers. Below are the most frequently asked SQL interview questions, with short, clear answers you can revise quickly.

1. What is the difference between INNER JOIN and LEFT JOIN?

INNER JOIN: returns only matching rows from both tables.

LEFT JOIN: returns all rows from left table + matching rows from right; if no match, right side is NULL.

SELECT c.customer_name, o.order_id
FROM Customers c
INNER JOIN Orders o
    ON c.customer_id = o.customer_id;
SELECT c.customer_name, o.order_id
FROM Customers c
LEFT JOIN Orders o
    ON c.customer_id = o.customer_id;

2. What is the difference between WHERE and HAVING?

  • WHERE filters rows before grouping/aggregation.
  • HAVING filters groups after aggregation (used with GROUP BY).
SELECT department, COUNT(*) AS emp_count
FROM Employees
WHERE status = 'Active'
GROUP BY department
HAVING COUNT(*) > 5;

3. How do you find duplicate records in a table?

Use GROUP BY and HAVING:

SELECT email, COUNT(*) AS cnt
FROM Users
GROUP BY email
HAVING COUNT(*) > 1;

4. Write a query to get the second highest salary.

One common method:

SELECT MAX(salary) AS second_highest_salary
FROM Employees
WHERE salary < (SELECT MAX(salary) FROM Employees);

Using ROW_NUMBER / DENSE_RANK (if supported):

WITH cte AS (
    SELECT salary,
           DENSE_RANK() OVER (ORDER BY salary DESC) AS rn
    FROM Employees
)
SELECT salary
FROM cte
WHERE rn = 2;

5. What is a PRIMARY KEY and FOREIGN KEY?

Primary Key: Uniquely identifies each row in a table. No duplicates, no NULLs. Foreign Key: A column that refers to the primary key of another table to maintain relationship.

6. What is the difference between DELETE, TRUNCATE and DROP?

  • DELETE – removes rows one by one (can use WHERE), can be rolled back.
  • TRUNCATE – removes all rows quickly, cannot use WHERE, usually not logged row-by-row.
  • DROP – removes the entire table structure + data.

7. What are aggregate functions in SQL?

Aggregate functions operate on a set of values and return a single value.

  • COUNT()
  • SUM()
  • AVG()
  • MIN()
  • MAX()

8. Explain GROUP BY with an example.

GROUP BY groups rows that have the same values into summary rows.

SELECT department, COUNT(*) AS emp_count
FROM Employees
GROUP BY department;

9. What is a subquery? Give an example.

A subquery is a query inside another query.

SELECT name, salary
FROM Employees
WHERE salary > (
    SELECT AVG(salary)
    FROM Employees
);

10. What are window functions?

Window functions perform calculations across a set of table rows that are related to the current row. Common examples: ROW_NUMBER(), RANK(), DENSE_RANK(), LAG(), LEAD(). 

SELECT employee_id,
       salary,
       RANK() OVER (ORDER BY salary DESC) AS salary_rank
FROM Employees;

11. How do you find employees who do not have any orders?

Use LEFT JOIN with NULL check:

SELECT e.employee_id, e.name
FROM Employees e
LEFT JOIN Orders o
    ON e.employee_id = o.employee_id
WHERE o.employee_id IS NULL;

12. What is the difference between UNION and UNION ALL?

  • UNION → combines result sets and removes duplicates.
  • UNION ALL → combines result sets and keeps duplicates.

13. How do you get top N rows per group?

Using window functions:

WITH ranked AS (
    SELECT
        customer_id,
        order_id,
        order_amount,
        ROW_NUMBER() OVER (PARTITION BY customer_id ORDER BY order_amount DESC) AS rn
    FROM Orders
)
SELECT *
FROM ranked
WHERE rn = 1;  -- top order per customer

14. How to find total sales per month?

SELECT
    DATE_TRUNC('month', order_date) AS month,
    SUM(order_amount) AS total_sales
FROM Orders
GROUP BY DATE_TRUNC('month', order_date)
ORDER BY month;

(Function name may differ by database: DATE_TRUNC, FORMAT, TO_CHAR, etc.)

15. How do you handle NULL values in SQL?

Use COALESCE or functions like ISNULL() / IFNULL() depending on DB.

SELECT COALESCE(phone_number, 'Not Provided') AS phone
FROM Customers;
✔ When preparing, type queries in a real SQL editor (MySQL, PostgreSQL, SQL Server, etc.) ✔ Focus on SELECT, JOIN, GROUP BY, HAVING, WHERE, subqueries, window functions.

These SQL questions are enough to clear most Data Analyst, MIS, BI, and Reporting interviews at the beginner to intermediate level. Practice writing the queries yourself and try using sample datasets.
